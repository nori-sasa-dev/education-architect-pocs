## 6. 各アプリ 仕様概要

### App①：スキル棚卸しチャットボット
**概要：** ユーザーが経験をAIと会話しながら整理し、スキル一覧を自動生成するツール

| 項目 | 内容 |
|------|------|
| UI | Streamlit |
| AI | Claude API（Anthropic） |
| データ保存 | JSON / CSV |
| 開発環境 | Cursor + Python |
| 用途 | ポートフォリオ用（将来的には業務ツール展開も検討） |
| コードベース | `pocs/career_ai/`（既存） |

**解決する課題：**
- スキルの言語化ができていない（特に非認知能力・暗黙知）
- キャリアの節目に「自分の強みが言えない」問題
- 管理職が部下のスキルをファクトベースで把握できない

### App⑥：暗黙知継承プラットフォーム（MVP）
**概要：** ベテランの知識をAIインタビューで引き出し、構造化してDBに蓄積。後継者が検索・活用できるシステム

| 項目 | 内容 |
|------|------|
| UI | Streamlit |
| AI | Claude API |
| データ保存 | SQLite（ローカルDB） |
| 開発環境 | Cursor + Python |
| ターゲット（MVP） | **NTTドコモソリューションズ社内**（A案を採択） |
| MVP後の拡張 | 音声対応、マルチユーザー、ベクトル検索 |

**解決する課題：**
- 団塊ジュニア世代の退職によるスキル・ノウハウの消失
- マニュアルに書けない「判断の理由・肌感覚」が継承されない
- 若手PMが「誰に聞けばいいかわからない」問題

## 8. 課題仮説の高リスク前提（要検証）

### App①
- 会話形式（チャット）のスキル棚卸しは有効な手法か
- CSV出力は実用的に活用されるか
- 継続して複数回使ってもらえるか（一回限りで終わらないか）

### App⑥
- AIとの対話形式で暗黙知を引き出せるか
- 抽出された知識を後継者が実際に検索・活用できるか
- 社内導入の承認が得られるか（情報セキュリティ・利用規約）
- ローカル環境（社内PC）での動作が求められるか
