| 項目 | 内容 |
|------|------|
| 場所 | `pocs/skill_inventory_bot/` |
| UI | Streamlit（ポート8501） |
| AI | Claude API（claude-sonnet-4-6）+ デモモード |

**ディレクトリ構成：**

├── agents/

- セッションリセット機能
- デモモード / 本番モード自動切替

**起動コマンド：**
```bash
cd pocs/skill_inventory_bot
source .venv/bin/activate
streamlit run app.py
```

## 6. 技術的な注意点（次回開発者向け）

### 環境構築
- Python 3.12 + venv（各アプリ独立）
- Streamlit設定：`~/.streamlit/config.toml` と `credentials.toml` が設定済み（headless=true, email=""）
- 依存パッケージ：`streamlit>=1.30.0`, `anthropic>=0.40.0`, `python-dotenv>=1.0.0`

### APIキー設定手順
1. https://console.anthropic.com/ でAPIキーを取得
2. 各プロジェクトで `.env.example` → `.env` にコピー
3. `ANTHROPIC_API_KEY=sk-ant-xxxxx` を記入
4. アプリ再起動で自動的に本番モードに切り替わる

### SQLite（App⑥）
- DBファイル：`tacit_knowledge_platform/data/knowledge.db`（自動生成）
- FTS5は日本語非対応のため不採用。LIKE検索を使用
- テスト時に生成されたデモデータが含まれている可能性あり（必要に応じて削除）

### デモモードの仕組み
- `InterviewAgent` / `ExtractionAgent` の `is_demo_mode` プロパティで判定
- APIキーが未設定の場合、`DEMO_RESPONSES` リストから固定応答を返す
- デモ応答は4〜5ターンで完了し、最終ターンでスキル/知識のJSON抽出マーカーを含む

### launch.json（Claude Code Preview用）
- `.claude/launch.json` に両アプリのPreview設定を記載済み
- `skill-bot`（ポート8501）、`tacit-knowledge`（ポート8502）

## 7. session02との関連

session02では「スキルの言語化は入口に過ぎない。本当に作りたいのは自己探索・ビジョン形成の体験」という問い直しが行われた。今回のMVP実装はsession01時点の仕様（スキル棚卸し / 暗黙知継承）に基づいている。

**今後の方向性として検討すべき点：**
- App①のシステムプロンプトを「スキル言語化」から「自己探索・ビジョン形成」へ拡張する可能性
- App⑥の「暗黙知をそのまま継承」するアプローチのリスク（レガシー固定化）への対応
- session02で発見された「安心感 × 問いかけ × 言語化」の要素をAI対話に組み込む
