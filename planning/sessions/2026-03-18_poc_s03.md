## 2. 決定事項

| # | 決定内容 | 理由・背景 |
|---|---------|-----------|
| 1 | APP①のコンセプトを「スキル棚卸しコーチ」→「自己探索コーチ」に転換 | Session02の気づき「スキル言語化は入口に過ぎない。本当に作りたいのは自己探索・ビジョン形成の体験」を反映 |
| 2 | APP①のアプリ名を「キャリア探索AI」に変更 | 自己探索体験を前面に出すため |
| 3 | マーカーを[SKILLS_EXTRACTED]→[DISCOVERY_COMPLETED]に変更 | スキル抽出から自己探索の完了へ意味を転換 |
| 4 | アウトプット構造をスキル一覧→自己探索サマリーに変更 | values, strengths, skills, direction, discovery_summaryの5要素構成 |
| 5 | APP⑥の編集・削除はアイテム単位（entry単位の一括削除UIは将来） | 最小限の工数で実用性を確保 |
| 6 | APP⑥の改善はdSOL実績作りとして位置づけ | Session02のリスク指摘「レガシーのやり方を焼き付ける」を認識した上での判断 |

## 3. 実装・作成した成果物

### APP① キャリア探索AI（旧：スキル棚卸しチャットボット）

**変更内容：**
- SYSTEM_PROMPT全面書き換え（自己探索コーチ、4フェーズ対話フロー）
- DEMO_RESPONSES 6件を新フローに合わせて書き換え
- `extract_skills()` → `extract_discovery()` にリネーム
- app.pyのUI文言変更（タイトル、キャプション、サイドバー、チャット入力プレースホルダー）
- サイドバー表示を新JSON構造（価値観・強み・スキル・方向性・サマリー）に対応
- export.pyを `discovery_to_json()` / `discovery_to_csv()` に変更

**対話フロー（4フェーズ）：**
1. Phase 1: 安心感の構築（ターン1-2）
2. Phase 2: 経験の深掘り（ターン3-5）
3. Phase 3: 価値観・強みの発見（ターン5-7）
4. Phase 4: 方向性の統合（ターン7-8）

### APP⑥ 暗黙知継承プラットフォーム

**変更内容：**
- db.pyに3メソッド追加：`update_knowledge_item()`, `delete_knowledge_item()`, `delete_knowledge_entry()`
- 検索ページに編集・削除機能を追加（`_render_knowledge_item()`関数）
- 編集：st.formによるインライン編集（タイトル・カテゴリ・内容・活用場面・キーワード）
- 削除：2段階確認（「削除」→「本当に削除する」）
- state管理：`editing_item_id`, `deleting_item_id` で編集/削除モードを制御
