## 2. 決定事項

| # | 決定内容 | 理由・背景 |
|---|---------|-----------|
| 1 | APIモデル名を `claude-sonnet-4-5-20250929` に統一 | テスト時のモデル名エラーを経て、動作確認済みのモデル名に統一 |
| 2 | APP①の問いかけが多すぎる問題 → 新POCで解決する方針 | APP①を修正するのではなく、フィードバックを活かした新POCとして設計 |
| 3 | マルチエージェントのリレー形式を採用 | 1体が延々と聞くのではなく、フェーズごとにエージェントが交代。「あなただけの特別チーム」という体験 |
| 4 | オーケストレーターは決定論的ステートマシン | LLM判定ではなく、フェーズ+ターン数+HANDOFFマーカーで遷移。デモモードの予測可能性とAPIコスト削減のため |
| 5 | セッション履歴はSQLiteで保存 | APP⑥のパターンを踏襲。UPSERT方式で発言のたびに自動保存 |
| 6 | 各エージェントのターン数を最大2-3に制限 | ユーザー発言回数4-5回（APP①の半分以下）で完結するセッション設計 |

### Career Team AI（APP④）

**ディレクトリ構成：**
```
pocs/career_team/
├── app.py                  # メインStreamlitアプリ（履歴保存機能付き）

│   ├── scout_agent.py      # 市場インテリジェンス（1-2ターン）
│   ├── strategist_agent.py # キャリア戦略参謀（1-2ターン）

├── orchestrator.py         # 5フェーズのステートマシン
├── database/

├── data/

│   └── export.py           # JSON/CSVエクスポート
```

**セッションフロー（5フェーズ）：**
1. Phase 1: OPENING（📣 Coach） — チーム紹介 + 最初の問いかけ
2. Phase 2: EXPLORATION（🪞 Mirror） — 価値観・強みの発見
3. Phase 3: MARKET（🔭 Scout） — 市場情報の提供
4. Phase 4: STRATEGY（♟️ Strategist） — キャリア方向性の提案
5. Phase 5: ACTION（📣 Coach） — アクションプラン策定 + レポート生成
