# 軌跡（Trace）— 持ち味発見ツール POC

過去複数期のシート（業績目標・行動目標・キャリアデザイン＋案件履歴）を**線で読み解き**、
点では見えなかった「その人らしい持ち味」をAIが発見して可視化する社内向けHRテックPOC。

## コンセプト

- **評価ツールではない**：査定・採点・優劣判定はしない。持ち味の「発見」に専念する
- **点 → 線**：1期分（点）では「たまたま」に見える行動も、複数期を貫くと核・芽・状況依存の持ち味として浮かぶ
- **所有権は社員**：抽出結果は本人のもの。上長と共有する範囲は本人が選ぶ
- **承認 ≠ 評価**：キャリアデザイン面談に乗せ、未来の活かし方を対話する道具として使う

## エンゲージメントの考え方

`(①言語化 × ②承認 × ③発揮) × Will方向舵`
本ツールは①言語化と②承認の橋渡しを担う。

## 持ち味の3タイプ

| タイプ | 意味 |
|--------|------|
| 🪨 核 | 複数期で繰り返し現れる、環境を問わない土台 |
| 🌱 芽 | 期を追って強まってきた成長中の持ち味 |
| ✨ 状況依存 | 特定の場面で際立つ「輝く条件」 |

## セットアップ

```bash
cd pocs/strength_trace
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # APIキーを設定（未設定ならデモモードで動作）
streamlit run app.py
```

## デモモード

`ANTHROPIC_API_KEY` 未設定時は、佐々木さん向けの固定3カードを表示する。
実APIモードでは選択した社員の各期シートをClaudeが分析する。

## ディレクトリ

```
├── data/sample_employees/      # 架空社員（佐々木・井上）各4期分

```

## ステータス

POC（プロトタイプ）。検証の主眼は「AI抽出に本人が"発見"を感じるか」。
DBは意図的に持たない（評価インフラ化の回避）。
